# 차트로드 자동 발행 및 SEO 글 구성 기준

## 발행 원칙

- 모든 글은 미리 작성해두고, `publishedAt` 시간이 지난 글만 사이트에 노출합니다.
- 하루 발행 수는 3~5개로 배정합니다.
- 기본 발행 슬롯은 오전 7~10시, 오후 12~15시, 저녁 18~21시에 각각 1개씩입니다.
- 추가 1~2개는 위 세 시간대 중 하나에 랜덤하게 배치합니다.
- 공개 전 글은 목록, 검색, 결과 추천, 관련글, RSS, 사이트맵에서 제외합니다.

## 글 구성 기준

각 글은 검색 유입과 학습 목적을 동시에 만족하도록 아래 순서로 작성합니다.

1. 검색 의도에 맞는 제목
   - 예: `엘리엇파동 기본 구조: 5파 상승과 3파 조정 이해`
   - 사용자가 실제로 검색할 표현을 제목 앞쪽에 둡니다.
2. 첫 문단
   - 이 글에서 무엇을 배울 수 있는지 2~3문장으로 명확히 설명합니다.
3. 선행 개념
   - 이해에 필요한 이전 글을 자연스럽게 연결합니다.
   - 예: `고점과 저점 구조를 먼저 이해하면 엘리엇파동의 위치 판단이 쉬워집니다.`
4. 핵심 설명
   - 개념 정의, 차트에서 확인하는 순서, 흔한 실수, 무효화 기준을 포함합니다.
5. 교육 이미지
   - 글의 핵심 판단 기준을 실제 차트 느낌의 이미지로 보여줍니다.
   - 이미지 안 문구는 한국어로 작성하고, 장식용 이미지보다 판단 구조를 우선합니다.
6. 체크리스트
   - 독자가 실제 차트에서 바로 확인할 수 있는 질문 3~5개를 넣습니다.
7. 내부링크
   - 본문 문장 안의 자연스러운 단어에 연결합니다.
   - 별도 링크 목록보다 문맥형 링크를 우선합니다.

## 내부링크 구조

- 허브 글: 넓은 주제의 공부 순서를 안내합니다.
- 클러스터 글: 캔들, 거래량, 이동평균선, 지지저항, 리스크 관리처럼 중간 주제를 묶습니다.
- 세부 글: 특정 기법, 조건, 무효화 기준, 실전 예시를 다룹니다.
- 결과 유형 글: 초보, 초중급, 중급, 고급 유형별로 먼저 읽을 글을 추천합니다.

내부링크는 `허브 -> 클러스터 -> 세부 글 -> 관련 세부 글 -> 허브` 형태로 순환하게 만듭니다.

## 주요 검색 키워드 묶음

- 차트 기초: 차트 보는 법, 캔들 보는 법, 양봉 음봉, 시가 고가 저가 종가
- 가격 구조: 고점 저점, 추세 구조, 지지선 저항선, 돌파 이탈
- 이동평균선: 이동평균선 보는 법, 20일선 60일선, 정배열 역배열, 이평선 눌림
- 거래량: 거래량 보는 법, 거래량 해석, 거래량 증가, 거래량 감소
- 리스크 관리: 손절 기준, 손익비 계산, 포지션 크기, 매매 금지 구간
- 고급 기법: 엘리엇파동, 피보나치, 다이버전스, 와이코프, 다우이론

## 자동화 사용 기준

- 새 글을 추가할 때는 `src/data/articles.ts`에 글 데이터를 추가합니다.
- 발행일은 자동 배정되므로 수동으로 날짜를 넣지 않습니다.
- 배포는 GitHub 예약 작업이 하루 3번 확인하되, 새로 공개될 글이 생긴 경우에만 Cloudflare Pages 재빌드를 유도합니다.
- 긴 글 생성이나 이미지 생성은 자동 작업에 넣지 않습니다. 비용과 사용량이 커질 수 있기 때문입니다.
